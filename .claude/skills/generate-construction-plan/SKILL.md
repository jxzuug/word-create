# 施工组织设计 / 服务方案 Word 文档生成

## 工作流程

### 步骤 1: 接收项目信息

用户会直接提供项目信息。典型格式：

```
项目名称：XXX
项目概况/服务内容：XXX
章节结构：
第一章、XXX
第二章、XXX
...
需要输出：XX页 或 约XXX字
```

**无需逐项询问** — 用户给什么就用什么。如果用户信息不完整，仅针对缺失的关键信息进行追问（如没有项目名称或章节结构时）。

### ⚠️ 严禁内容重复（最重要的规则）

生成内容时**绝对禁止出现重复段落**。同一句话、同一段话在文档中只能出现一次。

**常见的重复场景（必须避免）：**

| 场景 | 错误做法 | 正确做法 |
|------|---------|---------|
| 凑字数 | 把同一段话反复写多遍 | 每段内容必须唯一，宁可内容精炼 |
| 跨章节重复 | 各章节都写相同的"我公司将严格遵守..." | 每个章节写本节的特定内容，通用承诺只在总述写一次 |
| 首段模板化 | 每章开头都是"本项目是XXX，我公司将提供优质服务" | 每章直接切入主题，不重复项目介绍 |
| 末尾堆砌 | 文档末尾加几段"人员管理是...质量管理是...安全管理是..."的车轱辘话 | 末尾章节写完整后自然结束 |

**自检方法（生成完成后必须执行）：**

1. 逐段检查：任意两段文字含义或表述相似 → 删除重复，只保留一段
2. 检查文档末尾：如果末尾有大量看起来"似曾相识"的段落 → 全部删除
3. 如果感觉"好像在哪写过" → 这就是重复，删掉

**记住：用户一眼就能看出来重复内容，这是最严重的质量问题。宁可内容少而精，也绝不允许重复。**

### 步骤 2: 生成文档内容

根据用户提供的章节结构，逐章生成专业内容。

**内容质量标准（必须同时满足以下所有要求）：**

| 要求 | 说明 | 检查标准 |
|------|------|---------|
| **描述清晰** | 每段话逻辑清楚、条理分明，不使用模糊表述 | 读完后能明确知道"做什么、怎么做、标准是什么" |
| **内容具体** | 有具体数据、指标、方法名称、规范编号，不泛泛而谈 | 比如写质量标准要写"GB 50300-2013"而不是"国家规范" |
| **内容全面** | 从多角度、多层次展开，覆盖该主题的各个方面 | 管理措施应包含组织、制度、流程、检查、整改全链条 |
| **内容完整** | 每个章节有头有尾，有总述有分述有小结，不自相矛盾 | 表格完整、编号连续、引用有出处 |
| **内容翔实丰富** | 综合运用段落、表格、列表、流程图（用表格模拟）等多种表达方式 | 每章至少含1个表格和1个列表 |
| **图文结合** | 使用表格模拟流程图/组织架构图/流程图/饼图等视觉效果 | 见下方"图文结合示例" |

**图文结合示例（用表格模拟图形）：**

组织架构图用表格模拟：
```
┌─────────────────────────────────────────┐
│              项目经理                     │
├──────────┬──────────┬──────────────────┤
│ 技术负责人 │ 施工员    │ 安全员           │
├──────────┼──────────┼──────────────────┤
│ 电工班    │ 水暖班   │ 综合维修班        │
└──────────┴──────────┴──────────────────┘
```

→ 用表格实现：headers: ["技术负责人", "施工员", "安全员"], rows: [["电工班", "水暖班", "综合维修班"]]

流程图用表格模拟：
```
┌──────┐   →   ┌──────┐   →   ┌──────┐   →   ┌──────┐
│ 报修  │       │ 派单  │       │ 维修  │       │ 验收  │
└──────┘       └──────┘       └──────┘       └──────┘
```
→ 用表格实现：一行多列，每列一个步骤

**内容生成要点：**

- **严格遵循用户指定的章节标题和顺序**，一字不差
- **每章内容原创且不重复** — 每章的每段话必须唯一，禁止跨章节复制
- **内容充实**：每章至少 3-5 段实质性内容，不能只有标题
- **专业规范**：使用规范的行业术语
- **格式丰富**：适当使用表格、列表、分段来组织内容
- **紧扣项目**：内容要针对用户的项目特点，不能是通用模板（如用户项目是零星维修，就不能写成建筑施工）

**不同类型项目的内容重点：**

| 项目类型 | 内容侧重点 |
|---------|-----------|
| 建筑施工 | 施工工艺、技术措施、质量安全、进度计划、资源配置 |
| 维修改造/零星维修 | 响应机制、人员配置、故障处理、快速反应、服务质量 |
| 服务采购项目 | 服务方案、管理团队、服务承诺、应急保障、售后服务 |
| 设备安装 | 安装工艺、调试方案、技术参数、验收标准 |

### 步骤 3: 按子章节字数目标生成内容（关键方法）

**核心思路：每个子章节（二级标题）写 5000-6000 字，用子章节数量来达到总字数目标。**

```
总字数目标 ÷ 5000 = 需要写的子章节数量
```

操作方法：

**1. 如果用户指定了总字数或总页数：**

先换算：`总字数 = 总页数 × 550`（每页约550汉字）

然后按章节分配字数：
- 每个 **子章节**（二级标题）写 **5000-6000 字**
- 计算需要多少个子章节：`子章节数 = 总字数 ÷ 5000`
- 将子章节合理分配到各章（一级标题）中
- 每章至少1个子章节，重点章节多分配

**2. 如果用户没有指定字数：**

默认每个子章节写 5000-6000 字，根据章节复杂程度调整：
- 重点章节（施工技术措施、质量保证措施等）：2-3个子章节，共10000-18000字
- 一般章节：1个子章节，5000-6000字
- 简单章节：1个子章节，3000-4000字

**3. 每个子章节如何写足5000字：**

```
┌─────────────────────────────────────────┐
│ 二级标题（如"一、质量管理体系"）        │  ~50字
│ 段落1：总体介绍该子章节内容              │  ~500字
│ 段落2：具体措施/方法1                    │  ~500字
│ 段落3：具体措施/方法2                    │  ~500字
│ 表格：相关数据/要求对照表                │  ~300字
│ 段落4：具体措施/方法3                    │  ~500字
│ 段落5：实施要点/注意事项                 │  ~500字
│ 段落6：检查验收标准/效果评价             │  ~500字
│ ...                                      │
│ 合计                                     │  ~5000字
└─────────────────────────────────────────┘
```

**4. 关键规则：**

- **每个子章节的内容必须完全独立、绝不重复** — 从不同角度/层面展开
- 如果发现某个子章节写不够字数 → 增加更深层次的细节内容，不要往回找补
- 如果用户给的章节结构里没有二级标题 → 在content中使用heading level 2来划分子章节
- **字数只是参考，质量优先** — 写不够就如实说，**绝不通过重复来凑**

### 步骤 3b: 处理超大型文档（100页以上 / 50000字以上）

对于超大型文档，Claude 无法一次生成全部内容。必须采用 **分批增量写入** 的方式。

**Token 成本参考（提前告知用户）：**

| 规模 | 页数 | 约需轮数 | 总Output | 总Input | 合计 |
|------|------|---------|---------|---------|------|
| 小型 | ~10页 | 1轮 | ~5.5K | ~3K | ~8.5K |
| 中型 | ~50页 | 2轮 | ~27K | ~6K | ~33K |
| 大型 | ~100页 | 4轮 | ~55K | ~12K | ~67K |
| 超大型 | ~500页 | 19轮 | ~275K | ~57K | ~332K |
| 巨型 | ~1000页 | 37轮 | ~550K | ~111K | ~661K |

> 注：1页 ≈ 550汉字 ≈ 550 tokens。按每轮生成3章（~15K tokens）估算。

**优化策略（减少Token消耗）：**

| 策略 | 做法 | 效果 |
|------|------|------|
| 增大每轮批次 | 每轮生成5-6章而非2-3章 | 轮数减半，总token省30-40% |
| 精简已有内容回读 | 回读时只读章节标题和字数，不读全文 | 每轮input减少50-70% |
| 一次成型 | 如果100页内，尽量一轮完成全部章节 | 避免回读开销 |

**操作方法：**

```
第1轮：生成第1-3章的内容 → 写入 temp.json
第2轮：读取已有章节摘要 → 生成第4-6章 → 追加到 temp.json
第3轮：读取已有章节摘要 → 生成第7-9章 → 追加到 temp.json
...
最后：node generate-docx.js temp.json output.docx
```

**具体步骤：**

1. 先计算好总章节数和批次划分，告知用户需要分几轮、大致token消耗
2. 每轮生成3-6章的内容，**追加**写入同一个 JSON 文件
3. 每轮开始时读取已有 JSON 的章节标题（用摘要方式，不读全文）以避免跨轮重复
4. 所有章节完成后，调用脚本生成最终 .docx

**JSON 文件追加写入方法：**

```javascript
// 第1轮：创建初始文件
const data = { projectName: "...", subtitle: "...", chapters: [...] };
fs.writeFileSync('temp.json', JSON.stringify(data, null, 2));

// 第2轮及以后：读取、追加、写入

```

- 每轮生成时要回读已写的内容，避免跨轮重复
- 各轮之间的章节内容要衔接自然
- 生成全部完成后调用脚本生成 .docx
- **不要每轮都生成一个完整的 .docx** — 只在全部完成后生成一次
```

### 步骤 4: 构建 JSON 数据

组装为以下 JSON 格式：

```json
{
  "projectName": "XX项目",
  "subtitle": "施工组织服务方案",
  "companyName": "XXXX（如有）",
  "date": "2026年6月",
  "targetWordCount": 8000,
  "targetWordsPerSection": 5000,
  "outputName": "输出文件名",
  "chapters": [
    {
      "title": "第一章、施工管理目标",
      "level": 1,
      "content": [
        { "type": "paragraph", "text": "正文内容...", "format": { "firstLineIndent": 2 } },
        { "type": "heading", "text": "子标题", "level": 2 },
        { "type": "list", "items": ["项1", "项2"], "listType": "bullet" },
        { "type": "table", "headers": ["列1", "列2"], "rows": [["A", "B"]] },
        { "type": "pageBreak" }
      ]
    }
  ]
}
```

**content 单元类型：**

| type | 必填 | 说明 |
|------|------|------|
| `paragraph` | `text` | 段落，支持 `format.firstLineIndent`(缩进字符)、`format.bold`、`format.alignment`(left/center/both) |
| `heading` | `text`, `level` | 子标题，level 2-3 |
| `list` | `items`(string[]) | 列表，`listType`: "bullet"序号 / "number"数字 |
| `table` | `headers`, `rows` | 表格，表头自动加蓝底 |
| `image` | `base64`, `width`, `height` | 图片，base64编码（不含data:前缀），width/height为像素 |
| `pageBreak` | — | 分页符 |

**image 类型说明：**
```json
{ "type": "image", "base64": "iVBORw0KGgo...", "width": 500, "height": 300, "caption": "图1-1 管理体系流程图" }
```
- `base64`：图片的 base64 编码数据（不含 data:image/xxx;base64, 前缀）
- `width` / `height`：图片在文档中的显示尺寸（像素）
- `caption`（可选）：图片标题，显示在图片下方

## 文档格式规范

- **纸张**: A4 (210×297mm)，页边距各 2.54cm
- **页眉**: 右侧项目名称
- **页脚**: 居中页码
- **封面**: 项目名称（大号）+ 副标题 居中
- **一级标题**: 黑体 16pt，深蓝
- **二级标题**: 黑体 14pt
- **正文**: 宋体 12pt，1.5倍行距，首行缩进2字符，两端对齐
- **表格**: 表头蓝底，内容居中
- **目录**: 封面后自动生成
